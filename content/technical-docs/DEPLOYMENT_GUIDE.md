# TAZROUT Smart Irrigation System
## Complete Hardware Deployment & Installation Guide

This guide outlines the complete, step-by-step process for deploying the Tazrout Smart Irrigation System onto the actual farm hardware. It covers everything from installing the server operating system to connecting the field ESP32 nodes, adhering strictly to the **Single-Server LAN Architecture**.

---

## Phase 1: The Central Server Setup
The central server handles everything: MQTT Broker, Database, AI Engine, and the Backend. This machine is kept on the farm and requires no internet access once installed.

### 1.1 OS Installation
1. Prepare a PC or a Raspberry Pi 4 (8GB RAM recommended).
2. Install **Ubuntu 22.04 LTS** (or Debian) for maximum stability.
3. Once booted, update the system packages:
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

### 1.2 Installing Core Dependencies
Run the following commands to install the required system infrastructure:

**1. Install PostgreSQL (Database):**
```bash
sudo apt install postgresql postgresql-contrib -y
```

**2. Install Mosquitto (MQTT Broker):**
```bash
sudo apt install mosquitto mosquitto-clients -y
```

**3. Install Java 17 & Maven (For Spring Boot Backend):**
```bash
sudo apt install openjdk-17-jdk maven -y
```

**4. Install Python 3.10 & Pip (For AI Engine):**
```bash
sudo apt install python3 python3-pip -y
```

## Phase 2: Configuration

### 2.1 Database Configuration (PostgreSQL)
We must create the database for the backend.
1. Enter the PostgreSQL console: `sudo -i -u postgres psql`
2. Run the following SQL commands:
   ```sql
   CREATE DATABASE tazrout_db;
   CREATE USER tazrout_user WITH PASSWORD 'tazrout_pass123';
   GRANT ALL PRIVILEGES ON DATABASE tazrout_db TO tazrout_user;
   ALTER DATABASE tazrout_db OWNER TO tazrout_user;
   \q
   ```

### 2.2 Broker Configuration (Mosquitto)
By default, Mosquitto doesn't allow external connections. We must configure it to listen on the LAN and use passwords.
1. Create a password file:
   ```bash
   sudo mosquitto_passwd -c /etc/mosquitto/passwd tazrout_admin
   # Enter password when prompted (e.g., tazrout123)
   ```
2. Edit the config file: `sudo nano /etc/mosquitto/mosquitto.conf`
3. Add these lines at the bottom:
   ```conf
   listener 1883 0.0.0.0
   allow_anonymous false
   password_file /etc/mosquitto/passwd
   ```
4. Restart Mosquitto: `sudo systemctl restart mosquitto`

## Phase 3: Deploying the Application Code

Clone or copy the `TAZROUT-Dashboard` repository to the server.

### 3.1 Spring Boot Backend
The backend manages historical data and the WebSocket bridge.
1. Navigate to the backend folder: `cd TAZROUT-Dashboard/backend`
2. Open `src/main/resources/application.properties` and verify the settings:
   ```properties
   # Database
   spring.datasource.url=jdbc:postgresql://localhost:5432/tazrout_db
   spring.datasource.username=tazrout_user
   spring.datasource.password=tazrout_pass123

   # MQTT
   mqtt.broker-url=tcp://localhost:1883
   mqtt.username=tazrout_admin
   mqtt.password=tazrout123
   ```
3. Build and run the server:
   ```bash
   mvn clean package -DskipTests
   java -jar target/dashboard-0.0.1-SNAPSHOT.jar
   ```
   *(Note: The database tables like `sensor_readings` and `zones` will be auto-generated by Hibernate upon successful launch.)*

### 3.2 AI Engine (Mr. Salah's Python Script)
1. Navigate to the AI folder (wherever the AI scripts are located).
2. Install Python dependencies:
   ```bash
   pip3 install -r requirements.txt
   ```
3. Ensure the AI is configured to use the `tazrout_admin` credentials to connect to `localhost:1883`.
4. Run the engine:
   ```bash
   python3 ai_engine.py
   ```

### 3.3 What to REMOVE
Before connecting hardware, **you must ensure the simulator is dead.**
1. Locate `mock_data/mqtt_simulator.py`.
2. Do not run this file. If it is running, kill it (`Ctrl+C` or `pkill -f mqtt_simulator.py`). It will conflict with real hardware data.

## Phase 4: Connecting the Hardware & Dashboard

### 4.1 The Hardware (ESP32 Nodes & LoRa Gateway)
1. Ensure the ESP32 code and the Gateway code are flashed with the correct MQTT credentials (`tazrout_admin` / `tazrout123`).
2. Ensure the Gateway code targets the IP address of the Central Server (e.g., `192.168.1.100`) rather than `localhost`, since the Gateway might be a separate Raspberry Pi or module.
3. Power on the field nodes.
4. **Verification:** Open a terminal on the server and listen:
   ```bash
   mosquitto_sub -t "tazrout/#" -u tazrout_admin -P tazrout123 -v
   ```
   You should immediately see the hardware sending real JSON payloads to `tazrout/zones/X/sensors`.

### 4.2 The Flutter Dashboard
The Flutter dashboard is installed on the farm manager's Windows PC.
1. Build the release version on a Windows machine:
   ```bash
   cd TAZROUT-Dashboard/frontend
   flutter build windows
   ```
2. Copy the resulting folder (`build/windows/x64/runner/Release/`) to the desktop of the farm manager's PC.
3. Run `tazrout_dashboard.exe`.
4. If the PC is on the same Wi-Fi/LAN as the Central Server, the dashboard will connect via WebSocket, load historical analytics from the PostgreSQL DB, and display live data from the ESP32 nodes.

---

## Phase 5: Verification & Safety Testing
1. **Emergency Stop Test:** From the Flutter dashboard, press the **Emergency Stop** button. Check the `mosquitto_sub` terminal; you should instantly see `tazrout/system/emergency/stop` published. The hardware should immediately close all valves.
2. **Log Verification:** 
   - Backend logs: Look at the console running the `.jar` file.
   - Frontend logs: Located in a file named `tazrout.log` right next to the `tazrout_dashboard.exe` file on the Windows PC.
